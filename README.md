# Madison File Reviews

A web application that helps client managers review scraped file directories for completeness. Browse Files.com directories and run AI-powered gap analysis to detect missing agendas, minutes, and other government documents.

## Prerequisites

- **Node.js 20+** (for building the frontend)
- **Python 3.12+** (for running the backend)
- **Files.com API key**
- **Anthropic API key**

## Local Setup

### 1. Clone and install frontend dependencies

```bash
cd C:\wamp\www\madison-file-reviews
npm install
```

### 2. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure environment variables

Copy the example env file and add your API keys:

```bash
cp .env.example .env
```

Edit `.env` with your keys:

```
FILES_API_KEY=your-files-com-api-key
FILES_BASE_URL=https://app.files.com/api/rest/v1
ANTHROPIC_API_KEY=your-anthropic-api-key
```

### 4. Build the frontend

```bash
npm run build
```

This compiles the React app into the `dist/` folder, which the Python backend serves as static files.

### 5. Run the application

```bash
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

Open **http://localhost:8000** in your browser.

## Development Mode

For active development, run the frontend dev server and Python backend separately:

**Terminal 1 — Python backend:**
```bash
uvicorn app:app --host 0.0.0.0 --port 3001 --reload
```

**Terminal 2 — Vite dev server (with hot reload):**
```bash
npm run dev
```

Open **http://localhost:3000** — the Vite dev server proxies `/api` requests to the Python backend on port 3001.

## How It Works

1. **Browse** — Navigate your Files.com directories using the folder tree sidebar and breadcrumb bar. No need to type paths.
2. **Analyze** — Select a directory and click "Run Analysis." The app collects the full file listing and sends a structured summary to Claude.
3. **Review** — Claude examines the file listing for gaps and inconsistencies (missing date ranges, low file counts, missing file types) and returns a severity-graded report with recommendations.

## Azure App Service Deployment

This app is designed for Azure App Service (Python 3.12):

1. Build the frontend locally: `npm run build`
2. Deploy `app.py`, `requirements.txt`, `dist/`, and `data/` to Azure
3. Set the environment variables (`FILES_API_KEY`, `ANTHROPIC_API_KEY`) in Azure App Service Configuration
4. Azure will detect `requirements.txt` and install dependencies automatically
5. Set the startup command to: `uvicorn app:app --host 0.0.0.0 --port 8000`

Alternatively, use the included `Dockerfile`:

```bash
docker build -t madison-file-reviews .
docker run -p 8000:8000 --env-file .env madison-file-reviews
```
